# Handoff: Конструктор курса (Course Builder) — WordPress LMS plugin

## Overview
Экран **конструктора курса** для преподавателя в админке WordPress-плагина LMS (внутреннее имя — FS LMS). Преподаватель собирает структуру курса: **модули → уроки → шаги**. Слева — дерево курса с drag-and-drop уроков, справа — редактор выбранного урока с рядом «шагов» (как в Stepik). Каждый шаг имеет свой тип (Лекция, Видео, Практика, Тест, Файл) и свой редактор.

Референсы, на которые опирался дизайн: **stepik.org** (ряд шагов-плиток, дерево уроков слева, редактор справа) и плагин **Tutor LMS** + нативная админка WordPress (тёмное боковое меню, admin bar, postbox, кнопки).

## About the Design Files
Файлы в этом пакете — **дизайн-референсы, созданные в HTML/CSS/vanilla JS**. Это прототип, демонстрирующий внешний вид и поведение, **а не продакшн-код для прямого копирования**.

Задача — **воссоздать этот дизайн в реальном окружении плагина**: это WordPress-плагин, поэтому реализация — это PHP-шаблоны страницы админки (`add_menu_page`/`add_submenu_page`), enqueue стилей и скриптов через `wp_enqueue_style`/`wp_enqueue_script`, и фронтенд на том стеке, который принят в плагине. Конструктор интерактивный — логичнее всего собрать его как изолированное JS-приложение (React/Preact/Vue или vanilla, по принятому в проекте), монтируемое в контейнер на странице админки, с сохранением через WP REST API или AJAX (`admin-ajax.php`) + nonce.

Используйте нативные классы WordPress, где возможно (`.wrap`, `.button`, `.button-primary`, `.postbox`, `.notice`) — это снизит объём кастомного CSS и обеспечит согласованность с админкой.

## Fidelity
**High-fidelity (hifi).** Финальные цвета, типографика, отступы, иконки и интерактив. Воссоздавать пиксель-в-пиксель, но переиспользуя нативные стили админки WP там, где они совпадают (кнопки, postbox, меню — это и есть нативный WP-вид).

## Step chips (плитки шагов) — ключевой элемент, как в Stepik
- `.step-chip`: ширина 56px, вертикально иконка-бокс + подпись типа.
- `.step-chip-box`: 56×52px, радиус 9px, фон `#f0f1f3`, граница `1.5px transparent`. В левом верхнем углу номер шага (10px, 700). В центре — SVG-иконка типа (22×22).
- **Цвет по типу** (`data-type`): иконка окрашена в цвет типа.
- **Активная плитка**: фон = цвет типа (заливка), иконка и номер белые, граница `currentColor`, подпись типа окрашена и 700.
- **Hover**: `translateY(-2px)` + тень.
- **Плитка «+ Добавить»** (`.step-add`): белый фон, пунктирная граница `1.5px #2271b1`, синий «+».
- Плитки **перетаскиваются** (reorder внутри урока).

`.step-head`: цветной бейдж «Шаг N: {Тип}» (фон = цвет типа, белый текст, иконка 14px) + инпут названия шага (max 280px) + справа icon-кнопки «Дублировать» и «Удалить» (красная на hover).

## Rich-text toolbar (для Лекции)
`.rte` рамка + `.rte-toolbar` (фон `#f6f7f7`): селект стиля абзаца (Абзац / Заголовок 1 / Заголовок 2 / Цитата), затем кнопки B / I / U, списки (маркир./нумер.), цитата, ссылка, изображение, формула (Σ), таблица, исходный код — разделены `.rte-sep`. Это **мок** тулбара — в реальном плагине подключить редактор (TinyMCE через `wp_editor()`, или CKEditor как в Stepik, или Gutenberg-блоки).

## Interactions & Behavior
- **Выбор урока**: клик по уроку в дереве → подсветка, breadcrumb и редактор обновляются, активным становится первый шаг урока.
- **Сворачивание модуля**: клик по каретке или по строке модуля.
- **Drag-and-drop уроков**: HTML5 drag (`draggable`). При `dragover` показывается индикатор вставки сверху/снизу (`box-shadow` `.drop-before`/`.drop-after`). Урок можно перемещать **внутри модуля и между модулями**. После drop — пересчёт нумерации.
- **Drag-and-drop шагов**: reorder плиток внутри урока.
- **Добавить шаг**: клик по «+» открывает **поповер** (`.step-popover`) со списком 5 типов (иконка + название + описание). Выбор типа добавляет шаг и делает его активным.
- **Дублировать / удалить шаг**: icon-кнопки в шапке шага. Нельзя удалить единственный шаг (показывается toast).
- **Добавить урок / модуль**: кнопки внизу дерева. Новый урок добавляется в текущий модуль с одним шагом-Лекцией.
- **Редактирование названия урока**: инпут, при вводе обновляет дерево в реальном времени.
- **Toast**: всплывающее уведомление снизу по центру (тёмное `#1d2327`, авто-скрытие ~1.8s) на действия (перемещено, добавлено, удалено, сохранено).
- **Hover-состояния**: grip-иконки появляются на hover строк; плитки приподнимаются; кнопки темнеют.

## State Management
Модель данных (см. `course-builder.js`, объект `course`):
```
course = {
  title: string,
  modules: [
    {
      id, title, collapsed: bool,
      lessons: [
        {
          id, title, published: bool,
          steps: [ { id, type: 'lecture'|'video'|'practice'|'quiz'|'file', title, ...content } ]
        }
      ]
    }
  ]
}
```
Глобальное UI-состояние: `activeLessonId`, `activeStepId`, ephemeral `stepContent[stepId]` (HTML лекции). 
В реальном плагине: загрузка курса через REST/AJAX при монтировании, дебаунс-автосейв изменений (название урока, контент шага, порядок) с nonce. Каждый шаг — отдельная сущность (custom post type или своя таблица), уроки и модули — иерархия (meta/таблица связей с полем `order`).

### Typography
- Шрифт UI: системный стек WP — `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Oxygen-Sans, Ubuntu, Cantarell, "Helvetica Neue", sans-serif`. Базовый размер 13px.
- Моноширинный (код, эталонное решение): `IBM Plex Mono` (или системный mono).
- `<h1>` 23px/400; заголовок урока 20px/600; названия модулей/уроков 13px; лейблы-капс 11–12px/700 uppercase letter-spacing .04em.
### Radius / Shadow / Spacing
- Радиусы: кнопки/инпуты 3–5px, карточки 3–6px, плитки шагов 9px, pill-бейджи 9–12px.
- Тень карточек: `0 1px 1px rgba(0,0,0,.04)`; builder `0 1px 2px rgba(0,0,0,.05)`; hover-плитка `0 4px 10px rgba(0,0,0,.1)`; поповер `0 12px 40px rgba(0,0,0,.2)`.
- Сетка builder: `320px 1fr`. Отступы контента 10–22px.
### Responsive
- ≤960px: меню сворачивается до иконок (36px), builder становится одноколоночным, дерево сверху (max-height 280px со скроллом).

## Assets
- Все иконки — **инлайновые SVG** (admin bar, меню, типы шагов, тулбар, кнопки). Внешних иконочных библиотек нет. В реальном плагине можно заменить на Dashicons (нативные иконки WP) где уместно.
- Шрифт IBM Plex Mono — Google Fonts (для mono-блоков). Можно заменить на системный mono или бандлить локально.
- Изображений-ассетов нет (миниатюра курса — CSS-градиент с буквами «Py»).
